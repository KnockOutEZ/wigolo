# Bookshelf API v2 Reference

The Bookshelf API provides RESTful endpoints for managing books, authors, and reading lists. All endpoints return JSON and require authentication via Bearer token.

## Base URL

```
https://api.bookshelf.example.com/v2
```

## Authentication

Include your API key in the `Authorization` header:

```
Authorization: Bearer sk_live_abc123def456
```

All requests without a valid token receive a `401 Unauthorized` response.

## Rate Limiting

The API enforces rate limits per API key:

| Plan       | Requests/min | Burst |
|------------|-------------|-------|
| Free       | 60          | 10    |
| Pro        | 600         | 50    |
| Enterprise | 6000        | 200   |

Rate limit headers are included in every response:

```
X-RateLimit-Limit: 600
X-RateLimit-Remaining: 594
X-RateLimit-Reset: 1713187200
```

## Common Response Format

All successful responses follow this envelope:

```json
{
  "data": {},
  "meta": {
    "request_id": "req_7f3a9b2c",
    "timestamp": "2026-04-15T10:30:00Z"
  }
}
```

Error responses use the same envelope with an `error` field:

```json
{
  "error": {
    "code": "not_found",
    "message": "Book with ID 999 does not exist.",
    "status": 404
  },
  "meta": {
    "request_id": "req_8e4b0c3d",
    "timestamp": "2026-04-15T10:30:01Z"
  }
}
```

## Endpoints

### Books

#### List Books

```
GET /books
```

Returns a paginated list of books.

**Query Parameters**

| Parameter | Type    | Default | Description                          |
|-----------|---------|---------|--------------------------------------|
| `page`    | integer | 1       | Page number                          |
| `limit`   | integer | 20      | Items per page (max 100)             |
| `sort`    | string  | `title` | Sort field: `title`, `year`, `rating`|
| `order`   | string  | `asc`   | Sort order: `asc` or `desc`          |
| `genre`   | string  | —       | Filter by genre slug                 |
| `q`       | string  | —       | Full-text search across title/author |

**Example Request**

```bash
curl -H "Authorization: Bearer sk_live_abc123def456" \
  "https://api.bookshelf.example.com/v2/books?genre=sci-fi&limit=2"
```

#### Get Book

```
GET /books/:id
```

Returns a single book by its numeric ID.

**Example Response**

```json
{
  "data": {
    "id": 42,
    "title": "Neuromancer",
    "author": {
      "id": 7,
      "name": "William Gibson",
      "bio": "American-Canadian speculative fiction writer."
    },
    "year": 1984,
    "genre": "sci-fi",
    "isbn": "978-0-441-56956-4",
    "rating": 4.2,
    "pages": 271,
    "description": "The sky above the port was the color of television, tuned to a dead channel.",
    "tags": ["cyberpunk", "ai", "hacking"],
    "created_at": "2025-01-10T08:00:00Z",
    "updated_at": "2026-03-22T14:15:00Z"
  }

```

#### Create Book

```
POST /books
```

Creates a new book record. Requires `write` scope.

**Request Body**

```json
{
  "title": "The Dispossessed",
  "author_id": 22,
  "year": 1974,
  "genre": "sci-fi",
  "isbn": "978-0-06-051275-2",
  "pages": 387,
  "description": "A brilliant physicist decides to take action.",
  "tags": ["utopia", "anarchism", "physics"]
}
```

**Response** — `201 Created`

```json
{
  "data": {
    "id": 253,
    "title": "The Dispossessed",
    "author": {
      "id": 22,
      "name": "Ursula K. Le Guin"
    },
    "year": 1974,
    "genre": "sci-fi",
    "isbn": "978-0-06-051275-2",
    "rating": null,
    "pages": 387,
    "created_at": "2026-04-15T10:35:00Z",
    "updated_at": "2026-04-15T10:35:00Z"
  }
}
```

#### Update Book

```
PATCH /books/:id
```

Partially updates a book. Only include the fields you want to change.

**Request Body**

```json
{
  "rating": 4.5,
  "tags": ["utopia", "anarchism", "physics", "award-winner"]
}
```

**Response** — `200 OK` with the full updated book object.

#### Delete Book

```
DELETE /books/:id
```

Permanently removes a book record. Returns `204 No Content` on success.

### Authors

#### List Authors

```
GET /authors
```

**Query Parameters**

| Parameter | Type    | Default | Description              |
|-----------|---------|---------|--------------------------|
| `page`    | integer | 1       | Page number              |
| `limit`   | integer | 20      | Items per page (max 100) |
| `q`       | string  | —       | Search by author name    |

#### Get Author

```
GET /authors/:id
```

Returns the author and a summary of their books.

**Example Response**

```json
{
  "data": {

    "name": "William Gibson",

    "born": 1948,

    "book_count": 12,

      { "id": 42, "title": "Neuromancer", "year": 1984 },
      { "id": 43, "title": "Count Zero", "year": 1986 },
      { "id": 44, "title": "Mona Lisa Overdrive", "year": 1988 }
    ]
  }
}
```

### Reading Lists

#### Create Reading List

```
POST /lists
```

**Request Body**

```json
{
  "name": "Summer 2026",
  "description": "Books to read this summer",
  "visibility": "public",
  "book_ids": [42, 108, 253]
}
```

#### Add Book to List

```
POST /lists/:list_id/books
```

```json
{
  "book_id": 77
}
```

Returns `200 OK` with the updated list.

#### Remove Book from List

```
DELETE /lists/:list_id/books/:book_id
```

Returns `204 No Content`.

## Error Codes

| Code              | Status | Description                         |
|-------------------|--------|-------------------------------------|
| `bad_request`     | 400    | Malformed request body or params    |
| `unauthorized`    | 401    | Missing or invalid API key          |
| `forbidden`       | 403    | Insufficient scope for this action  |
| `not_found`       | 404    | Resource does not exist             |
| `conflict`        | 409    | Duplicate ISBN or resource conflict |
| `rate_limited`    | 429    | Too many requests                   |
| `internal_error`  | 500    | Unexpected server error             |

## Pagination

All list endpoints support cursor-based pagination as an alternative to page numbers:

```
GET /books?cursor=eyJpZCI6NDJ9&limit=20
```

The `meta.pagination` object includes a `next_cursor` field when more results are available:

```json
{
  "meta": {
    "pagination": {
      "limit": 20,
      "has_more": true,
      "next_cursor": "eyJpZCI6NjJ9"
    }
  }
}
```

## Webhooks

You can register webhook URLs to receive events when resources change:

```
POST /webhooks
```

```json
{
  "url": "https://yourapp.example.com/hooks/bookshelf",
  "events": ["book.created", "book.updated", "book.deleted"],
  "secret": "whsec_your_signing_secret"
}
```

Each webhook delivery includes an `X-Signature` header for verification.
