# TypeError: Cannot read properties of undefined (reading 'map')

**Asked:** 3 years ago | **Modified:** 6 months ago | **Viewed:** 245k times

**Tags:** javascript, reactjs, arrays, undefined, typeerror

---

## Question

**Score: 127**

I'm getting the error `TypeError: Cannot read properties of undefined (reading 'map')` in my React component. My code fetches data from an API and tries to render a list:

```javascript
function UserList() {
  const [data, setData] = useState();

  useEffect(() => {
    fetch('/api/users')
      .then(res => res.json())
      .then(json => setData(json));
  }, []);

  return (
    <ul>
      {data.users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

The API returns `{ users: [...] }` but I get the error on the first render. What am I doing wrong?

---

## Accepted Answer

**Score: 198** | Answered by **javascript_expert**

The error occurs because on the initial render, `data` is `undefined` (the default value of `useState()` with no argument). When you try to access `data.users`, you're reading `.users` on `undefined`, which throws.

### Fix 1: Initialize state with the expected shape

```javascript
const [data, setData] = useState({ users: [] });
```

This ensures `data.users` is always an array, even before the API responds.

### Fix 2: Use optional chaining

```javascript
{data?.users?.map(user => (
  <li key={user.id}>{user.name}</li>
))}
```

### Fix 3: Conditional rendering

```javascript
{data && data.users && data.users.map(user => (
  <li key={user.id}>{user.name}</li>
))}
```

### Fix 4: Loading state (recommended)

```javascript
function UserList() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch('/api/users')
      .then(res => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then(json => {
        setData(json);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;
  if (!data?.users) return <p>No users found</p>;

  return (
    <ul>
      {data.users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

**Why Fix 4 is best:** It handles all three states (loading, error, success) and prevents the component from rendering before data is available. This pattern is standard in production React apps.

### Answer 2

**Score: 45** | Answered by **react_dev_42**

You can also use a custom hook to abstract the fetch logic:

```javascript
function useFetch(url) {
  const [state, setState] = useState({
    data: null,
    loading: true,
    error: null,
  });

  useEffect(() => {
    let cancelled = false;

    fetch(url)
      .then(res => res.json())
      .then(data => {
        if (!cancelled) {
          setState({ data, loading: false, error: null });
        }
      })
      .catch(error => {
        if (!cancelled) {
          setState({ data: null, loading: false, error: error.message });
        }
      });

    return () => { cancelled = true; };
  }, [url]);

  return state;
}

function UserList() {
  const { data, loading, error } = useFetch('/api/users');

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;

  return (
    <ul>
      {data.users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

The `cancelled` flag prevents state updates after the component unmounts, avoiding the "Can't perform a React state update on an unmounted component" warning.

### Answer 3

**Score: 23** | Answered by **ts_enthusiast**

If you're using TypeScript, you can make this error impossible at compile time:

```typescript
interface UserResponse {
  users: Array<{ id: string; name: string }>;
}

function UserList() {
  const [data, setData] = useState<UserResponse | null>(null);

  useEffect(() => {
    fetch('/api/users')
      .then(res => res.json())
      .then((json: UserResponse) => setData(json));
  }, []);

  if (!data) return <p>Loading...</p>;

  return (
    <ul>
      {data.users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

TypeScript will flag `data.users.map` as an error because `data` could be `null`. This forces you to add a null check before accessing the property.
